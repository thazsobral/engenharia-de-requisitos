# Engenharia de Requisitos

Este repositório contém materiais, estudos e documentação sobre **Engenharia de Requisitos**, um pilar fundamental no desenvolvimento de sistemas de software. O objetivo deste projeto é organizar conhecimentos essenciais para a elicitação, análise, especificação e validação de requisitos.

## 📚 Sobre o Projeto
A Engenharia de Requisitos é a área da Engenharia de Software responsável por entender as necessidades dos stakeholders e transformar esses desejos em requisitos técnicos claros e implementáveis. 

Este repositório serve como base de consulta para os principais conceitos, técnicas e processos da área, conforme detalhado no site do projeto: [https://thazsobral.github.io/engenharia-de-requisitos/](https://thazsobral.github.io/engenharia-de-requisitos/)

## 🏗️ Estrutura do Conteúdo
O conteúdo está organizado para cobrir o ciclo de vida dos requisitos:

* **Introdução à Engenharia de Requisitos:** Definições e importância.
* **Elicitação de Requisitos:** Técnicas para extrair informações dos usuários e clientes.
* **Análise de Requisitos:** Modelagem e negociação.
* **Especificação de Requisitos:** Escrita de documentos formais (como SRS).
* **Validação de Requisitos:** Garantindo que o produto construído é o produto desejado.
