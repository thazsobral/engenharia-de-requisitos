# Engenharia de Requisitos: Guia Definitivo

Bem-vindo ao repositório do **Guia Definitivo de Engenharia de Requisitos**. Este projeto foi concebido para ser uma plataforma educacional de referência, focada em fornecer uma visão estruturada, moderna e prática sobre uma das disciplinas mais críticas no ciclo de vida de desenvolvimento de software.

## 📚 Sobre o Projeto

Este site foi estruturado como uma **Single Page Application (SPA)**, priorizando a performance e uma experiência de usuário limpa, inspirada em padrões de documentação técnica de alta qualidade (estilo Stripe/Tailwind UI).

### Tecnologias Utilizadas
* **HTML5 Semântico:** Estrutura clara e acessível.
* **CSS3:** Design moderno, responsivo, com variáveis CSS para controle temático (Light/Dark Mode).
* **Vanilla JavaScript:** Lógica interativa, manipulação de DOM e gestão de estado do tema.

---

## 🏗️ Estrutura do Conteúdo

O site está organizado nas seguintes seções principais:

### 1. Introdução
Definição conceitual da Engenharia de Requisitos e sua importância vital no sucesso de projetos de software.
<img src="https://raw.githubusercontent.com/thazsobral/engenharia-de-requisitos/refs/heads/main/public/assets/images/ciclo.png" />

### 2. História e Evolução
Uma jornada cronológica, desde os processos tradicionais até as metodologias modernas.

### 3. Técnicas e Métodos
Cards interativos detalhando abordagens práticas para levantamento, análise e especificação.

### 4. Engenharia de Requisitos na Era da IA
Análise de como LLMs, automação e análise de sentimentos estão transformando a forma como capturamos e refinamos requisitos.

### 5. Protocolos de Trabalho
Tabela comparativa essencial para tomada de decisão:

| Critério | Protocolo Rigoroso (ISO/IEC 29148) | Protocolo Ágil |
| :--- | :--- | :--- |
| **Foco** | Documentação exaustiva | Valor de negócio/entrega |
| **Flexibilidade** | Baixa (Change Control rígido) | Alta (Backlog refinado) |
| **Ferramental** | Ferramentas de modelagem complexas | User Stories e Cards |

### 6. Playground Interativo
Uma dinâmica de aprendizado onde o usuário testa sua capacidade de classificar requisitos reais (Funcionais, Não-Funcionais e Regras de Negócio).

## 🚀 Como Executar

1. Clone este repositório:
   ```bash

   ```

2. Abra o arquivo `index.html` diretamente no seu navegador.
